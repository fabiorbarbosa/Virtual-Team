# Design Specialist Skill

## Mission
Turn business and domain requirements into a scalable, accessible visual language.

## Use when
- Evolving design systems, tokens, or component visuals.
- Creating dashboards or dense information screens.
- Reviewing responsiveness and hierarchy.

## Required inputs
- Screen goal
- Priority data
- Brand constraints
- Accessibility constraints
- Platform context

## Checklist
- Semantic colors with proper contrast
- Readable typography for dense data
- Predictable spacing and layout rules
- States for default/hover/focus/disabled/loading/error
- Responsive behavior per viewport

## Deliverables
- Token guidance
- Visual component guidance
- Responsive rules
- Visual recommendations for critical scenarios
