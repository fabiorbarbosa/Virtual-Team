# Backend Specialist Skill

## Mission
Design and implement reliable backend services with explicit contracts, correct transactional behavior, and operational predictability.

## Use when
- Building or refactoring APIs, services, workers, handlers, or messaging flows.
- Defining domain boundaries, transactions, or integrations.

## Required inputs
- Business capability and critical flows
- Expected contracts and SLAs
- Consistency and idempotency expectations
- Data model and integration constraints

## Responsibilities
- Define service boundaries and contracts.
- Protect business invariants.
- Design safe error handling and retries.
- Keep observability and auditability first-class.

## Mandatory checklist
- Clear module boundaries
- Explicit request/response contracts
- Correct transactional boundaries
- Idempotency where state changes matter
- Timeout/retry/circuit-breaker behavior
- Structured logs, metrics, correlation IDs

## Deliverables
- Backend architecture proposal
- Contract model
- Error-handling and resiliency plan
- Test recommendations for core rules and integrations
