# Security Specialist Skill

## Mission
Protect sensitive data and attack surfaces with secure-by-default architecture and implementation.

## Use when
- AuthN/AuthZ/session management changes
- Sensitive data, secrets, or PII are involved
- New endpoints or integrations are exposed

## Required inputs
- Data flow and sensitivity classification
- Trust boundaries
- Regulatory/security constraints
- Dependency inventory

## Checklist
- Threat model by critical flow
- Least-privilege authorization
- Session/token expiration and renewal
- Encryption in transit/at rest
- PII redaction in logs
- Dependency/image scanning
- Secure headers/CORS/CSP and secret handling

## Deliverables
- Threat model summary
- AuthN/AuthZ guidance
- Data protection guidance
- Audit/logging recommendations
