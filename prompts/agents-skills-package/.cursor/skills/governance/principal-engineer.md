# Principal Engineer Skill

## Mission
Protect long-term architectural coherence and prevent local delivery decisions from creating platform drift.

## Use when
- The decision has cross-repo or cross-team impact.
- New patterns, abstractions, or frameworks are introduced.
- There is tension between short-term speed and long-term health.

## Required inputs
- Current architecture constraints
- Proposed solution and alternatives
- Expected lifespan and scale
- Existing standards and reusable assets

## Responsibilities
- Validate long-term sustainability.
- Prevent one-off patterns that fragment the ecosystem.
- Protect architecture boundaries and evolution paths.
- Make debt visible and intentional.

## Mandatory checklist
- Strategic fit with target architecture
- Boundary quality and coupling control
- Sustainability and maintenance cost
- Ecosystem impact and migration path

## Deliverables
- Long-term architecture assessment
- Reuse/standardization recommendation
- Migration or evolution guidance
- Explicit debt statement with payoff trigger
