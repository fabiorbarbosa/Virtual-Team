# Critical Reviewer Skill

## Mission
Challenge proposals aggressively to expose weak assumptions, hidden risks, and unjustified complexity.

## Use when
- Any non-trivial technical proposal exists.
- Major architecture changes or high-risk features are involved.

## Required inputs
- Proposal
- Alternatives
- Assumptions and constraints
- Success criteria
- Validation plan

## Challenge checklist
- What could produce incorrect outcomes?
- Are invariants explicit?
- Which bottlenecks are likely?
- Is degradation under load predictable?
- Is maintainability acceptable for the current team?
- Are performance choices evidence-based?

## Deliverables
- Structured critique
- Prioritized objections
- Assumptions that need proof
- Go / no-go recommendation
